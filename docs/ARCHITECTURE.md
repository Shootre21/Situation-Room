# Architecture and File-by-File Guide

## Runtime Overview

The app is a client-rendered React SPA using Vite.

Core runtime loop:
1. Poll external APIs at intervals in live mode.
2. Normalize source-specific payloads into shared TypeScript models.
3. Render normalized entities on the globe.
4. Optionally browse historical snapshots via timeline playback.

## Main Files

### `src/main.tsx`
- React bootstrap / root mount.

### `src/App.tsx`
- Primary orchestration layer.
- Holds global UI/data state:
  - layers
  - feeds (flights, maritime, weather, satellites, earthquakes, geo events)
  - alerts
  - timeline/playback state
- Calls data services and snapshots history.

### `src/services/dataService.ts`
- All external HTTP fetch logic.
- Converts heterogeneous source formats into app-native models.
- Performs validation + dedupe.

### `src/components/GlobeView.tsx`
- 3D globe rendering and animation.
- Adds visual overlays:
  - points for flights/ships/satellites/weather/geo
  - rings for earthquakes
  - paths for GEO orbits
- Interpolates motion between data refresh cycles.

### `src/components/SidebarLeft.tsx`
- Layer toggles and controls.

### `src/components/SidebarRight.tsx`
- Alert/event stream presentation.

### `src/components/TopBar.tsx`
- High-level metrics + globe spin control.

### `src/components/BottomTimeline.tsx`
- Playback/live controls and timeline seeking.

### `src/types.ts`
- Shared interfaces for state + feed objects.

### `src/index.css`
- Global styling for the tactical UI presentation.

---

## State Boundaries

- **Data acquisition boundary:** `dataService.ts`
- **State orchestration boundary:** `App.tsx`
- **Visualization boundary:** `GlobeView.tsx`
- **Control surfaces:** top bar + sidebars + bottom timeline

This separation keeps API-specific logic out of rendering components.

---

## Rendering Strategy

- Points are generated by active layer filters and mapped to color/size semantics.
- Motion is projected by heading + speed + elapsed time from `fetchedAt`.
- GEO path overlays provide orbital context for geosynchronous satellites.
- High-resolution tile imagery improves visual fidelity when zoomed in.

---

## Error Handling Philosophy

- Source fetch failures degrade gracefully to empty arrays.
- One failing provider should not block rendering of other layers.
- Alerts are generated from available data only.

---

## Extension Checklist

When extending functionality:
1. Update types first.
2. Add/modify data fetchers.
3. Pipe state through `App.tsx`.
4. Render in `GlobeView.tsx` + controls as needed.
5. Document behavior in `README.md` and `docs/APIs.md`.
6. Run `npm run lint` and `npm run build`.
